# 貯金管理アプリ

Googleスプレッドシートと連携した、毎月の資産を管理・可視化するWebアプリです。
Google Apps Script としてデプロイし、ブラウザから使用します。

---

## セットアップ

### 1. Apps Script エディタを開く

1. 連携するGoogleスプレッドシートを開く
2. メニュー「**拡張機能**」→「**Apps Script**」をクリック

### 2. ファイルを貼り付ける

**Code.gs**（既存ファイルを置き換え）
1. 左側の `コード.gs` を選択
2. 中身をすべて削除し、`Code.gs` の内容を貼り付け

**Index.html**（新規ファイルを追加）
1. 左側の「**＋**」→「**HTML**」を選択
2. ファイル名を `Index` と入力（`.html` は不要）
3. 中身をすべて削除し、`Index.html` の内容を貼り付け

### 3. デプロイする

1. 右上「**デプロイ**」→「**新しいデプロイ**」
2. 歯車アイコン ⚙️ →「**ウェブアプリ**」を選択
3. 以下のように設定して「デプロイ」をクリック

| 項目 | 設定値 |
|---|---|
| 次のユーザーとして実行 | 自分 |
| アクセスできるユーザー | 自分のみ（または任意） |

4. 初回はGoogleアカウントの権限承認が必要（下記参照）
5. 発行された **ウェブアプリURL** をブックマークしてアクセス

### 4. 権限を承認する（初回のみ）

「このアプリは確認されていません」と表示された場合：

1. 「**詳細**」をクリック
2. 「**〇〇（安全でないページ）に移動**」をクリック
3. 「**許可**」をクリック

### 5. コードを更新したとき

「デプロイ」→「**デプロイを管理**」→ 鉛筆アイコン → バージョンを「**新バージョン**」にして「デプロイ」

## 画面の説明

### ダッシュボード

最新月の資産状況をひと目で確認できます。

| 表示項目 | 内容 |
|---|---|
| 総資産 | 現金 ＋ 投資の合計 |
| 現金総額 | 銀行・財布・キャッシュレスの合計 |
| 投資総額 | 各投資口座の合計 |
| 前月比較 | 先月との差額（プラスは緑・マイナスは赤） |
| 年間貯金額 | 今年の累計増加額 |

下部に直近12ヶ月の資産推移グラフを表示します。

### データ入力

毎月末に各口座の残高を入力し、スプレッドシートに保存します。

1. 「**対象月**」で入力する月を選択
2. 各項目に残高を入力（空欄はスキップされます）
3. 「**スプレッドシートに保存**」をクリック

> **ヒント**: 既存の月を選ぶと、その月のデータが自動で読み込まれます。

### グラフ

| グラフ | 内容 |
|---|---|
| 総資産推移 | 全期間の総資産・現金・投資の折れ線グラフ |
| 現金 vs 投資 | 毎月の内訳を積み上げ棒グラフで表示 |
| 月間増減 | 毎月の増減額（増加は緑・減少は赤） |
| 銀行口座推移 | 各銀行口座の残高推移 |
| 投資口座推移 | 各投資口座の残高推移 |

### データ一覧

直近12ヶ月のデータを一覧で確認できます。

- 上部のボタンでカテゴリを絞り込めます
- `合計` 行は背景色が変わります

### ⚙ 設定（項目の追加・削除）

管理する項目をカスタマイズできます。

**項目を追加する**
1. 追加したいカテゴリの入力欄に項目名を入力
2. 「**＋ 追加**」をクリック
3. スプレッドシートに新しい行が追加され、データ入力画面にも反映されます

**項目を削除する**
1. 削除したい項目の「**削除**」ボタンをクリック
2. 確認ダイアログで「OK」を押すと削除されます

> ⚠️ 削除するとスプレッドシートの行ごと削除されます。その月のデータも失われます。

**新しいカテゴリを追加する**
1. 設定画面下部の「新しいカテゴリを追加」フォームに入力
2. カテゴリ名と最初の項目名を入力して「**カテゴリを追加**」をクリック

**削除できない行**
`計算式` バッジが付いている行（〇〇合計・現金総額・全合計など）は、スプレッドシートの計算式が入っているため削除できません。

## コードの説明

### Code.gs（サーバーサイド）

Google Apps Script のサーバーサイドコードです。スプレッドシートの読み書きを担当します。
ブラウザ（Index.html）から `google.script.run` 経由で各関数が呼び出されます。

#### 定数

```javascript
const SPREADSHEET_ID = '...';  // スプレッドシートのID（URLから取得）
const SHEET_NAME = '貯金推移'; // 対象シート名
```

#### `doGet()`

```javascript
function doGet() { ... }
```

ウェブアプリのエントリポイント。URLにアクセスされたとき `Index.html` を返します。
Apps Script をウェブアプリとしてデプロイすると、アクセス時にこの関数が自動で実行されます。

#### `getFullData()`

```javascript
function getFullData() { ... }
```

スプレッドシートの**全データ**と**行構造**を1回のAPI呼び出しでまとめて返します。

- `data`: 全セルの値（2次元配列）。Date型は `yyyy/MM/dd` 文字列に変換して返します
- `structure`: 各行のカテゴリ・ラベル・計算式かどうかの情報（後述の `buildStructure_` が生成）

ページ読み込み時に1度だけ呼ばれ、以降はフロントエンドがこのデータを使って画面を構築します。

#### `buildStructure_(sheet, lastRow, lastCol)`（内部関数）

```javascript
function buildStructure_(sheet, lastRow, lastCol) { ... }
```

シートの**A列（カテゴリ）・B列（項目名）・C列の数式**を読み取り、各行の情報を配列として返します。

```javascript
// 返り値の例
[
  { row: 2, category: '銀行', label: '楽天銀行',  isInput: true,  isTotal: false },
  { row: 5, category: '銀行', label: '銀行合計',  isInput: false, isTotal: true  },
  ...
]
```

- `isInput: true` — ユーザーが値を入力する行
- `isTotal: true` — C列に `=SUM(...)` などの計算式が入っている行（削除・上書き不可）

A列はスプレッドシート上でセルが結合されているため、値があるセルの値を `currentCat` として引き継ぎ、空セルに補完します。

#### `addItem(category, label)`

```javascript
function addItem(category, label) { ... }
```

指定したカテゴリに新しい項目行を挿入します。

挿入位置のロジック：
- **既存カテゴリ**の場合 → そのカテゴリの合計行（`isTotal: true`）の直前に挿入。合計行がなければ最終行の次
- **新カテゴリ**の場合 → 「現金総額」「投資総額」などの集計サマリー行の直前に挿入

`sheet.insertRowBefore(row)` で行を挿入することで、既存の `SUM` 計算式の範囲が自動的に拡張されます。

#### `deleteItem(rowIndex)`

```javascript
function deleteItem(rowIndex) { ... }
```

指定した行番号（1始まり）のシート行を削除します。
行1（ヘッダー）への削除は安全のためエラーにします。

#### `saveMonthData(monthStr, inputData)`

```javascript
function saveMonthData(monthStr, inputData) { ... }
```

指定月のデータをスプレッドシートに書き込みます。

- `monthStr`: `'yyyy/MM'` 形式の文字列（例: `'2024/03'`）
- `inputData`: `{ 行番号: 値, ... }` のオブジェクト（例: `{ "2": 294165, "3": 148418 }`）

**既存月**の場合は対応する列を検索して上書きします。
**新しい月**の場合は右端に列を追加し、前月の計算式が入っているセルを `copyTo()` でコピーします（相対参照が自動調整されるため `SUM` 範囲もズレません）。

### Index.html（フロントエンド）

HTML・CSS・JavaScript をひとつのファイルにまとめたシングルページアプリです。
`google.script.run` を使ってサーバー側の関数を非同期で呼び出します。

#### グローバル変数

```javascript
let allData   = null;  // スプレッドシートの全データ（2次元配列）
let structure = [];    // 行構造（buildStructure_ の返り値と同じ形式）
let months    = [];    // 月リスト [{ idx: 列インデックス, label: 'yyyy/MM' }]
const charts  = {};   // Chart.js インスタンスの保持（再描画時に destroy() するため）
```

#### 初期化フロー

```
window.onload
  └─ google.script.run.getFullData()
        └─ init(raw)
              ├─ allData / structure / months を設定
              ├─ buildDashboard()   ダッシュボード描画
              ├─ buildInputForm()   入力フォーム構築
              ├─ buildCharts()      グラフ描画
              ├─ buildTable()       データ一覧構築
              └─ buildSettings()    設定タブ構築
```

ページ読み込み時に `getFullData()` を1度呼ぶだけで全画面の初期化が完了します。
データを更新（保存・追加・削除）した後は `reload()` が `getFullData()` を再実行して全画面を更新します。

#### `val(rowDef, colIdx)`

```javascript
function val(rowDef, colIdx) { ... }
```

`allData[rowDef.row - 1][colIdx]` の値を数値で返すユーティリティ関数。
空セル・null・undefined は `0` として扱います。

#### `findRow(label)` / `findRows(cat)`

```javascript
function findRow(label) { return structure.find(r => r.label === label); }
function findRows(cat)  { return structure.filter(r => r.category === cat); }
```

`structure` からラベルまたはカテゴリで行定義を検索します。
ダッシュボードで「全合計」「現金総額」などの行を動的に参照するために使います（行番号のハードコードを避けるため）。

#### `buildDashboard()`

最新月（`months` 配列の末尾）のデータを取り出し、カードとグラフを描画します。
直近12ヶ月分を Chart.js の折れ線グラフで表示します。

#### `buildInputForm()`

`structure` の `isInput: true` の行だけを抽出し、カテゴリごとにフォームを動的生成します。
月選択（`<input type="month">`）が変わると `onMonthChange()` が呼ばれ、既存データをフォームに自動セットします。

#### `buildCharts()`

Chart.js を使って5種類のグラフを描画します。
`mkChart()` はグラフ生成の共通処理をまとめたヘルパー関数で、再描画時に既存インスタンスを `destroy()` してからnewします（メモリリーク防止）。

#### `buildSettings()`

`structure` を元に設定タブのUIを動的生成します。
`isTotal: true` の行は `計算式` バッジを表示し削除ボタンを出しません。
追加・削除後は `reload(() => buildSettings())` で画面全体を再構築します。

## 技術スタック

| 技術 | 用途 |

| Google Apps Script | サーバーサイド・スプレッドシート操作 |
